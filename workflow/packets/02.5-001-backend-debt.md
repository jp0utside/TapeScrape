## Goal

Close the four 🟡 important follow-ups from the Phase 2 boundary review before starting
Phase 3. All are backend-only, no new features, no API shape changes. Closes **F2-2**
(metadata cache regression), **F2-3** (silent IA failure), **F2-4** (divergent format
policies), **F2-5** (duplicated staleness decision). Also picks up the trivial **F2-6**
(ruff auto-fix) since it touches overlapping files.

## Acceptance criteria

- [ ] **F2-2 — Metadata cache restored.** `get_item_metadata` checks `MetadataCache`
      before hitting IA; on miss it fetches, caches with ~24 h TTL, then returns.
      `MetadataCache` is constructed in the FastAPI lifespan, stored on `app.state`,
      and injected via a `Depends` provider in `routes/deps.py` (same pattern as
      `get_ia_client`). The module-global `SearchCache` in `routes/search.py` is moved
      to `app.state` at the same time (closes the standing 🟢 note).
- [ ] **F2-3 — IA errors logged.** `orchestrate.py`'s per-item `except Exception`
      block logs the identifier and exception at `warning` level through the project
      logger (`core/logging`). A test asserts the log line is emitted on a simulated
      metadata-fetch failure.
- [ ] **F2-4 — One format policy, one layer.** `_PLAYABLE_FORMATS` in `aggregate.py`
      is removed. `models/ia.py` `_UNSUPPORTED_FORMATS` is replaced by
      `_PLAYABLE_FORMATS` — a positive allowlist of iOS-playable formats including
      bitrate-labelled MP3 variants (e.g. `"64Kbps MP3"`, `"128Kbps MP3"`,
      `"VBR MP3"`). The `drop_unsupported_formats` validator becomes `keep_playable_formats`
      and uses the allowlist. `aggregate.py:_build_tracks` trusts the parse layer and
      does not re-filter by format. A unit test confirms bitrate-MP3 files survive parse.
- [ ] **F2-5 — Single staleness owner.** The route (`routes/concerts.py`) no longer
      checks `get_aggregation_age` or decides staleness. It calls
      `aggregate_artist(artist, ia_client, force=False)` unconditionally and lets the
      orchestrator own the stale-or-cached decision. The route still wraps the call in
      `asyncio.wait_for` for the 504 timeout. Staleness logic lives in exactly one
      place (`orchestrate.py`).
- [ ] **F2-6 — ruff clean.** `ruff check --fix` produces zero findings on
      `backend/`. No behavior change.
- [ ] All existing backend tests pass. `pytest` default run makes zero live IA calls.
- [ ] No change to any HTTP response shape or client-visible behavior.

## Interface sketch

```python
# ia/metadata.py — now cache-aware
async def get_item_metadata(
    client: IAClient, identifier: str, cache: MetadataCache
) -> IAItem:
    cached = await cache.get(identifier)
    if cached is not None:
        return IAItem.model_validate(cached)
    response = await client.get(f"/metadata/{identifier}")
    data = response.json()
    await cache.set(identifier, data)  # default TTL = 86400 (24 h)
    return IAItem.model_validate(data)

# aggregation/orchestrate.py — cache threaded, exception logged
async def aggregate_artist(
    canonical_artist: str,
    ia_client: IAClient,
    metadata_cache: MetadataCache,
    force: bool = False,
) -> list[AggregatedConcert]:
    ...
    try:
        ia_item = await get_item_metadata(ia_client, identifier, metadata_cache)
        ...
    except Exception:
        logger.warning("metadata_fetch_failed identifier=%s", identifier, exc_info=True)
        continue

# routes/deps.py — new providers
def get_metadata_cache(request: Request) -> MetadataCache:
    return request.app.state.metadata_cache

def get_search_cache(request: Request) -> SearchCache:
    return request.app.state.search_cache

# routes/concerts.py — no staleness check; just call orchestrate
@router.get("", response_model=ConcertListResponse)
async def list_concerts(
    artist: str,
    page: int = Query(1, ge=1),
    ia_client: IAClient = Depends(get_ia_client),
    metadata_cache: MetadataCache = Depends(get_metadata_cache),
) -> ConcertListResponse:
    try:
        concerts = await asyncio.wait_for(
            aggregate_artist(artist, ia_client, metadata_cache, force=False),
            timeout=_AGGREGATION_TIMEOUT,
        )
    except asyncio.TimeoutError:
        ...

# models/ia.py — positive allowlist
_PLAYABLE_FORMATS = {
    "VBR MP3", "MP3", "64Kbps MP3", "128Kbps MP3",
    "Flac", "FLAC", "24bit Flac", "WAVE", "AIFF",
}

@field_validator("files", mode="before")
@classmethod
def keep_playable_formats(cls, files: list) -> list:
    return [f for f in files if f.get("format") in _PLAYABLE_FORMATS]
```

## Known ambiguities / open questions

- **Which bitrate-MP3 format strings exist in the wild?** IA uses at least `"64Kbps MP3"`
  and `"128Kbps MP3"`. The allowlist should cover known variants; if an unknown MP3
  variant appears later it will be silently dropped (same as today's denylist for unknown
  formats). Add any additional variants discovered during implementation. A future
  follow-up could switch to a regex (`r".*MP3$"`) if the long tail is problematic — not
  in this packet.
- **`MetadataCache` TTL.** The design doc says ~24 h; `MetadataCache.set` already
  defaults to `ttl_seconds=86400`. No change needed; note it's configurable if needed
  later.
