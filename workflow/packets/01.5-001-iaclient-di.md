## Goal

Replace the two module-level `IAClient()` singletons (`ia/search.py:4`,
`ia/metadata.py:4`) with a **single** `IAClient` constructed in the FastAPI lifespan and
injected via dependency injection, and fix the rate limiter so its lock is no longer held
across `await asyncio.sleep(...)` or the HTTP call. Closes Phase 1 review follow-ups
**F1-2** and **F1-5 / F0-2**. No change to any HTTP response shape.

This is the prerequisite the Phase 1 boundary review flagged "fix before Phase 2 parallel
aggregation": aggregation fans out concurrent Metadata calls, and two independent httpx
clients with two independent rate limiters defeat the single shared rate limiter.

## Acceptance criteria

- [ ] No module-level `IAClient()` instance exists anywhere. `grep -rn "IAClient()" backend`
      finds it only inside the FastAPI lifespan (and test fixtures).
- [ ] A single `IAClient` is created in `main.py`'s lifespan, stored on `app.state`, and
      `await client.aclose()` is called on shutdown.
- [ ] `search_items(...)` and `get_item_metadata(...)` take the `IAClient` as an explicit
      parameter; they hold no module/global client state.
- [ ] The `/concerts/{id}` route obtains the client through a FastAPI `Depends(...)`
      provider that reads `request.app.state.ia_client`, and the provider is overridable
      in tests via `app.dependency_overrides`.
- [ ] `IAClient.get`: the `asyncio.Lock` is held **only** for the slot-reservation
      bookkeeping. It is NOT held across `await asyncio.sleep(...)` nor across
      `await self._client.get(...)`. Requests remain spaced by `>= _min_interval`.
- [ ] A unit test drives N concurrent `IAClient.get()` calls against a stubbed httpx
      transport (no live IA) and asserts: (a) successive sends are spaced by
      `>= _min_interval`, (b) the calls are NOT fully serialized on the lock across the
      sleep (concurrency is preserved — total wall time ≈ `(N-1) * _min_interval`, not
      `N * (interval + request_time)`).
- [ ] All existing backend tests pass, updated for the new signatures. `pytest`
      default run makes zero live IA calls (existing `live_ia` markers untouched).
- [ ] The `/concerts/gd-1977-05-08` response body is byte-for-byte unchanged vs. before
      this packet (assert via an existing/expanded route test).

## Interface sketch

```python
# core/http_client.py — rate limiter (lock no longer spans sleep / HTTP call)
async def get(self, path, params=None, *, cache_hit=False) -> httpx.Response:
    async with self._lock:
        scheduled = max(time.monotonic(), self._last_request_at + self._min_interval)
        self._last_request_at = scheduled
    delay = scheduled - time.monotonic()
    if delay > 0:
        await asyncio.sleep(delay)          # outside the lock
    logger.info("ia_request url=%s%s cache_hit=%s", settings.ia_base_url, path, cache_hit)
    response = await self._client.get(path, params=params)   # outside the lock
    response.raise_for_status()
    return response

# main.py
@asynccontextmanager
async def lifespan(app: FastAPI):
    setup_logging()
    app.state.ia_client = IAClient()
    try:
        yield
    finally:
        await app.state.ia_client.aclose()

# routes/concerts.py
def get_ia_client(request: Request) -> IAClient:
    return request.app.state.ia_client

@router.get("/{concert_id}", response_model=ConcertResponse)
async def get_concert(concert_id: str,
                      ia_client: IAClient = Depends(get_ia_client)) -> ConcertResponse: ...

# ia/search.py / ia/metadata.py
async def search_items(client: IAClient, *, query=None, creator=None, date=None,
                       rows=50, page=1) -> IASearchResult: ...
async def get_item_metadata(client: IAClient, identifier: str) -> IAItem: ...
```
