## Deviations from packet

- The packet's Read-first noted the Phase-1 rate limiter "held the lock across
  `asyncio.sleep` + the HTTP call." On reading the shipped code, the lock spanned only
  `asyncio.sleep(wait)` — the HTTP call was already outside it. The fix still applies
  (the lock is now O(1) bookkeeping, sleep moved out too) and the concurrency test was
  retuned (HTTP delay ≫ rate interval) so it sharply guards the "lock across the HTTP
  call" regression the F1-5 wording describes, rather than relying on a marginal margin.

## Notes for review

`get_ia_client` deliberately lives in `routes/concerts.py`, not `core/` (CONVENTIONS §1:
`core/` imports only stdlib + Pydantic; the provider needs `fastapi.Request`). Promotion
to a shared `routes/deps.py` is deferred until a second Phase 2 route consumes it — no
abstraction before a second consumer. The module-scope `TestClient(app)` pattern does not
run the lifespan, so non-live route tests inject a dummy via `app.dependency_overrides`
(safe because IA calls are fully mocked there); the live test opts into the real lifespan
with `with TestClient(app)`. This DI + overridable-provider shape is the seam Phase 2
aggregation will build on; worth a CONVENTIONS entry at the Phase 2 review if it recurs.
