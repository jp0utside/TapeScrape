**Packet:** 04.5-001-download-restore-reconcile

## What was done

Closed F4-1: fixed the `DownloadManager` restore-path state desync and replaced the
`Task.sleep` restore race in tests with a deterministic `whenRestored()` hook.

### Changes

**`TapeScrape/Downloads/DownloadManager.swift`**

- Added `private var restoreTask: Task<Void, Never>?` property.
- Stored the restore task: `restoreTask = Task { await restoreState() }`.
- Added `func whenRestored() async { await restoreTask?.value }` — a fire-and-forget-safe
  hook that completes when restore is done; production callers never need it.
- Rewrote `restoreState()`: calls `rehydrateTaskMap()` first (repo mutations only), then
  does exactly **one** final `allDownloads()` read and seeds `recordingProgress` for every
  record — records with an active task get `.downloading(progress:)` computed from
  `completedTracks/totalTracks`; all others get `record.state` verbatim. This makes the
  mirror equal to the repository for every record after restore.
- Rewrote `rehydrateTaskMap()`: removed the mirror-seeding it previously did. Now handles
  only: task-map build via `getAllTasks` + `findTrackByStreamURL`, orphan cancellation,
  and marking `.downloading`-with-no-active-task records `.failed` in the repo. No longer
  seeds `recordingProgress` — that is now exclusively `restoreState()`'s single pass.

The two-reads-without-reconcile structure is gone. `rehydrateTaskMap()` runs first so its
interrupted→failed repo mutation is reflected in the single final `allDownloads()` read.

**`TapeScrapeTests/Downloads/DownloadManagerTests.swift`**

- Restructured all three existing tests to set up full repo state **before** creating the
  `DownloadManager` (correct relaunch simulation; eliminates the manager-creation/repo-
  mutation race). Replaced the restore-phase `Task.sleep(100ms)` in each with
  `await manager.whenRestored()`.
- The `Task.sleep(100ms)` after `retryDownload()` calls is retained (the retry inner Task
  has no clean hook; acceptable per packet scope).
- Added `restoreMirrorEqualsRepositoryAfterRestore()`: primes two records (`.downloaded`
  and `.failed`), creates a fresh manager, `await whenRestored()`, then asserts
  `recordingState(for:) == record.state` for each record — the exact F4-1 desync scenario.
