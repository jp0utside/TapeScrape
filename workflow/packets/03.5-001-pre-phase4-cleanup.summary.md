## What shipped

### F3-2 ‚Äî Shared SQLite connection with WAL and checked writes

- **NEW** `TapeScrape/Repositories/LibraryDatabase.swift` ‚Äî `actor LibraryDatabase` owns
  the single shared connection to `library.sqlite`. Opens with `PRAGMA journal_mode=WAL`
  and `PRAGMA busy_timeout=3000`. Runs schema creation for all tables (library + history)
  via static helpers in `init`. Closes connection in `deinit`. Exposes `nonisolated var
  pointer: OpaquePointer? { db }` so repos can capture it synchronously at init time ‚Äî
  no `await` required.

- `SQLiteLibraryRepository.swift` ‚Äî changed `init(dbURL:)` ‚Üí `init(database:)`;
  removed `sqlite3_open`/`sqlite3_close`/`deinit`; removed `createTables` static (moved
  to `LibraryDatabase`); kept `seedFavoriteTag` (repo-specific seed data); added
  `checkedStep(_:context:)` helper; updated `exec`, `addTag`, `createPlaylist`,
  `upsertSnapshot`, `insertPlaylistItem` to use checked step.

- `SQLitePlaybackHistoryRepository.swift` ‚Äî same structural changes (`init(database:)`,
  no open/close/deinit, no schema method); `recordPlay` now checks `sqlite3_step` return
  code and prints a warning on failure.

- `TapeScrapeApp.swift` ‚Äî constructs one `LibraryDatabase(url:)` and passes it to both
  repos.

- `SQLiteLibraryRepositoryTests.swift` ‚Äî `makeRepo()` updated to construct a
  `LibraryDatabase` first then `SQLiteLibraryRepository(database:)`.

- `SQLITE_TRANSIENT` stays file-local in both repo files per ¬ß18 convention (DRY only
  at 3 impls).
